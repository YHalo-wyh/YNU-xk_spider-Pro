# YNU选课助手 Pro v2.0.0

## 🎯 重大更新：批次自动识别，支持第三轮

本次版本聚焦“轮次兼容性”与“登录后自动准备流程”，核心目标是让程序在教务系统轮次切换后无需手动改代码也能继续使用。

---

## 🚀 核心更新

### 1. 选课批次自动识别
- 登录成功后，程序会优先从 `student/{studentCode}.do` 的学生信息中提取当前轮次。
- 若学生信息里未返回有效轮次，自动调用 `elective/batch.do` 接口兜底识别。
- 识别结果会作为全流程 `electiveBatchCode` 使用，不再写死第二轮批次常量。

### 2. 自动执行轮次确认（xklcqr）
- 与网页端逻辑对齐：识别到轮次后自动调用 `student/xklcqr.do` 完成轮次确认。
- 对“需先确认轮次才能查课/选课”的场景（第三轮常见）可直接兼容。
- 确认失败时不阻断登录，会在日志中给出提示并继续运行。

### 3. 登录界面批次展示优化
- 登录区批次文案改为“自动识别”。
- 登录成功后动态显示当前轮次（如：轮次名 + 批次码），便于排查问题。

---

## 🛠️ 修复与优化

- 修复：课程查询、监控、选课、退课等流程使用的 `electiveBatchCode` 固定为旧值的问题。
- 优化：README 与版本说明同步更新，明确“按当前开放轮次自动适配（含第三轮）”。

## 🧭 升级建议

1. 关闭旧版本程序（若在监控中，请先点击“停止”）。
2. 使用 v2.0.0 安装包升级，或解压便携版到新目录。
3. 首次登录后确认日志中已显示当前轮次信息，再开始监控。

---

## ⚠️ 注意事项

- 程序已支持按教务系统当前开放轮次自动识别（含第三轮）。
- 暂不支持跨专业课程、重修课程及辅修课程的选退。
- 若登录后仍无法获取课程，请反馈 4 个接口响应：
  - `student/{studentCode}.do`
  - `elective/batch.do`
  - `student/xklcqr.do`
  - 课程查询接口（如 `elective/recommendedCourse.do`）
