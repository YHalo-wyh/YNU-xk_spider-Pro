# YNU选课助手 Pro v2.1.0

## 🎯 稳定性修复版本：成功核实、匹配精度、日志清晰化

本次版本聚焦“结果可信”和“运行可读性”：减少误判成功、避免误退相似课程，并让监控停止相关日志可准确区分场景。

---

## 🚀 核心更新

### 1. 成功通知改为“核实后再提示”
- 抢课成功和换课成功均增加核实流程。
- 仅当已选课程查询明确返回“已选中”时，才输出成功通知。
- 查询失败或未选中时继续监控，不再提前报喜。

### 2. 冲突课程定位改为严格全名匹配
- 移除基于方括号内容和课程名前缀的模糊匹配。
- 优先按 `conflictDesc` 严格全名匹配，避免“课程名包含关系”导致误匹配。
- 典型修复场景：`计算机网络` 不再误命中 `计算机网络实践`。

### 3. 监控停止日志分类优化
- 手动点击停止：显示“监控已停止”。
- 退出登录：显示“已退出登录（监控已停止）”。
- 会话过期自动重登：显示“会话过期，监控已暂停”。
- 关闭程序：不再额外追加停止日志，避免混淆。

### 4. 选课批次改为纯自动识别 + 重试
- 登录后按接口自动识别当前批次并重试。
- 不再回退默认批次常量，避免错误批次导致的查课/选课异常。
- 多次识别失败时直接提示“批次自动识别失败，请稍后重试登录”。

## 🛠️ 修复与优化

- 修复：检查更新弹窗版本号显示可能出现 `vv2.x.x` 的问题。
- 修复：监控停止状态在日志中重复输出的问题。
- 优化：停止原因日志语义统一，便于问题排查和用户判断当前状态。

## 🧭 升级建议

1. 关闭旧版本程序（若在监控中，请先点击“停止”）。
2. 使用 v2.1.0 安装包升级，或解压便携版到新目录。
3. 登录后观察日志是否已输出当前批次信息，再开始监控。

---

## ⚠️ 注意事项

- 程序已支持按教务系统当前开放轮次自动识别（含第三轮）。
- 暂不支持跨专业课程、重修课程及辅修课程的选退。
- 若登录后仍无法获取课程，请反馈以下接口响应：
  - `student/{studentCode}.do`
  - `elective/batch.do`
  - `student/xklcqr.do`
  - 课程查询接口（如 `elective/recommendedCourse.do`）
