# YNU选课助手 Pro v2.3.0

## 🔐 安全与轮次兼容修复版本

本次版本聚焦安全修复与当前选课轮次兼容性：恢复 HTTPS 证书校验、避免 token 明文进入日志，并同步适配网页端当前使用的轮次查询方式。

---

## 🚀 核心更新

### 1. 恢复 HTTPS 证书校验
- 移除所有业务请求中的 `verify=False`。
- 登录、验证码、课程查询、选课、退课、结果核实等链路均恢复默认 TLS 校验。
- 降低公共网络、校园网代理、DNS 劫持等中间人场景下的账号密码和会话泄露风险。

### 2. 不再静默屏蔽 TLS 告警
- 移除 `urllib3.disable_warnings(InsecureRequestWarning)`。
- 保留 PyInstaller 打包环境下的 `certifi` 证书路径修复，确保正常证书链可以被校验。

### 3. 登录日志脱敏
- 登录成功后不再将 token 明文写入 UI 日志。
- 日志仅提示“Token 已获取”，避免日志文件泄露会话凭据。

### 4. 轮次兜底识别兼容当前网页端
- 轮次兜底接口 `elective/batch.do` 改为优先使用 GET，与当前网页端调用方式保持一致。
- 若旧服务端返回 GET 不支持，则自动回退 POST，尽量不影响历史环境兼容性。
- 第一轮、第二轮、第三轮等仍按教务系统返回的当前可用轮次自动识别。

### 5. 版本与构建同步升级
- 程序版本升级至 `v2.3.0`。
- 打包产物命名同步升级至 `v2.3.0`。

---

## 🛠️ 兼容性说明

- 本版本不改变课程查询、监控、抢课、换课、自动重登、Server酱通知等业务流程。
- 若个别网络环境存在证书链异常，请优先修复系统/网络证书配置，不建议关闭证书校验。

## 🧭 升级建议

1. 关闭旧版本程序（若在监控中，请先点击“停止”）。
2. 使用 `v2.3.0` 安装包升级，或解压便携版到新目录运行。
3. 登录后确认“选课批次”显示为教务系统当前开放轮次，再开始监控。
