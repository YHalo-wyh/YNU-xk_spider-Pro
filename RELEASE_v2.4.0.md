# YNU选课助手 Pro v2.4.0

## 🛡️ 安全换课与扩展通知增强版本

本次版本聚焦“监控并发真实生效、自动换课安全边界、升级数据隔离、开发者自定义通知、自动更新体验与 Watchdog 恢复一致性”。如果你会同时监控互相冲突的课程，建议仔细阅读“待抢冲突组策略”和“换课风险说明”。

---

## 🚀 核心更新

### 1. 真正有界的 HTTP 并发

- “HTTP 并发”现在会限制同时在途的网络请求数量。
- 每门课程仍独立监控，但查询请求会通过有界并发槽控制。
- 每个监控线程使用独立 `requests.Session`，避免多线程共享 Session 的状态竞争。
- 停止监控时记录实际 HTTP 并发峰值，便于排查并发配置是否生效。

### 2. 待抢列表内部冲突组策略

- 开始监控前自动扫描待抢列表内部的时间冲突。
- 默认策略：同组任意一门抢成功后，自动停止其它互相冲突的待抢课程。
- 每个冲突组可选择“首选课程”：非首选先抢到时，首选继续监控；首选后续有余量时可能触发自动换课。
- Watchdog 闪退恢复、Session 过期自动重登恢复会复用上次冲突组策略，不再重复弹窗。

### 3. 自动换课安全边界增强

- 冲突课程定位优先使用服务端 `conflictDesc` 严格全名匹配。
- 如果服务端描述不足，再使用目标课与已选课的时间交集做唯一匹配。
- 无法唯一定位冲突课程，或同时匹配到多门已选课程时，拒绝自动退课。
- 目标课选课接口返回成功但核实接口异常时，不再直接继续监控；会进入安全救援流程，先确认目标课是否已在课表中，确认不了则持续尝试抢回旧课。
- 回滚成功、回滚未确认等关键状态会写入日志，并可触发自定义 Webhook 事件。

### 4. 更新安装与本地数据隔离

- 账号密码、待抢课程、监控状态、Watchdog 信号与运行日志迁移到独立用户数据目录。
- 安装包不包含 `config.json`、`monitor_state.json`、`watchdog_signal.json` 或运行日志。
- 覆盖安装/升级不会覆盖本地账号、密码和待抢记录。
- 自动更新下载完成后，安装前会保存配置和待抢状态、停止监控与 Watchdog，再启动安装程序。

### 5. 更新检查与下载体验优化

- 检查更新使用后台 Worker，不阻塞界面，并显示“检查中”进度弹窗。
- GitHub API 请求增加 User-Agent、超时与重试。
- 发现可用安装包时，用户可直接下载并安装。
- 下载时显示进度条；服务器返回总大小时显示百分比，否则显示滚动进度和已下载大小。
- 当前线上 latest 低于本地版本时，会提示当前已是最新版本，不会误触发降级下载。

### 6. 开发者模式升级为完整 Webhook 通道

- 入口保留在「帮助 → 开发者模式」。
- “启用开发者模式”区域改为更明显的高亮大复选框。
- 支持多个自定义 Webhook 通道。
- 支持事件筛选、GET/POST/PUT/PATCH/DELETE 请求方法、Headers、URL Params、JSON/Form/Raw Body 模板。
- 支持测试发送，方便确认自建通知服务是否可用。
- 支持事件：`test`、`course_available`、`select_success`、`swap_success`、`rollback_success`、`rollback_failed`、`conflict_target_retired`。
- 旧版简单 URL 模板配置会自动兼容迁移为新版 GET 通道。

### 7. 登录错误与 Watchdog 源码模式修复

- 按云南大学登录接口返回信息识别账号/密码错误，并写入脱敏日志。
- 修复源码模式下 Watchdog 路径多退一层的问题，现在可正确定位项目根目录的 `run_watchdog.py`。
- 打包版 Watchdog 行为保持兼容。

## 🧭 升级建议

1. 升级前先关闭旧版本主程序和 Watchdog；如果正在监控，请先点击“停止”。
2. 推荐使用 `YNU.Pro_v2.4.0_Setup.exe` 覆盖安装。
3. 首次启动后确认账号、密码、待抢课程是否恢复正常。
4. 如果待抢列表存在互相冲突的课程，开始监控前请仔细选择默认策略或首选优先级。
5. 如果使用开发者通知，请进入「帮助 → 开发者模式」检查新版 Webhook JSON 配置，并先执行一次测试发送。

## ⚠️ 注意事项

- 自动换课本质上存在“先退旧课、再抢新课”的时间窗口；多人同时抢同一门课、网络异常、接口异常或旧课名额被其他人占用时，都无法保证抢课或回滚一定成功。
- 选择“首选课程”后，非首选课程先抢成功时，首选仍会继续监控；首选后续有余量可能触发再次换课，请根据课程重要程度谨慎选择。
- 冲突课程无法唯一定位时，程序会拒绝自动退课，这是刻意保守的安全策略。
- Webhook 配置可能包含 Token、密钥或内网地址，请不要公开截图或分享完整配置文件。
- 本地账号密码和待抢记录已迁移到用户数据目录，不再建议把安装目录当作数据备份目录。
